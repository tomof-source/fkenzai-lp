## 1. v3 から v4 への変更点（実装）

### 1-1. コピー改修（カモシダ案＋ユカリ修正）

- タイトル：「夕方、西日で部屋がサウナになる家。なぜ年々、ひどくなるのか。」→ **「夕方、西日で部屋がサウナになる家。」**（句点二打ち削除でリズム改善）
- リード文末：「分かりやすくお話しします」削除＋「夏の窓問題」概念語削除
  - 旧：エアコンを強めても、夕方の西向きの部屋だけは蒸し風呂のようになる。多くの家庭で起きている「夏の窓問題」について、熊本と福岡で窓のリフォームを20年やってきた立場から、原因と対策を分かりやすくお話しします。
  - 新：エアコンを強めても、夕方の西向きの部屋だけは蒸し風呂のようになる。毎年6月になると、同じ悩みのご相談が増えはじめます。熊本と福岡で窓のリフォームを20年やってきた立場から、書きました。
- h-1 冒頭：「広告で見た悩みの続きが、ここに書いてある」と0.5秒で錯覚させるために、お客様の生の声から始める（岡村風指摘）
  - 旧：毎年6月に入ると、お客様から決まって同じご相談が増えはじめます。「夕方になると、リビングだけ妙に暑い」「2階の子ども部屋だけ、エアコンが効かない」。
  - 新：「夕方になると、リビングだけ妙に暑い」。「2階の子ども部屋だけ、エアコンが効かない」。毎年6月に入ると、決まって同じご相談が増えはじめます。…広告でこのページに辿り着いた方は、たぶん心当たりがあるはずです。
- h-4 見出し：「窓の外で日差しを止める、という考え方」→ **「窓の外で、日差しを止める」**（LP的擬似中立フレーズ「という考え方」削除）

### 1-2. 折りたたみ実装（読了時間の体感短縮）

- h-3「カーテンで遮るのが、なぜ効きにくいのか」の後半詳細（お湯と浴槽の比喩・遮熱フィルムの限界）を `<details class="fold">` で折りたたみ化
- 体感3分・実態8分の構成にする（シノミヤ「読了18-22%→25-28%伸ばす」狙い）

### 1-3. マイクロCV挿入（h-4 と h-5 の間）

- ボタン文言（カモシダ確定）：「うちの窓、対象になるか30秒で見る」
- サブ文言：「熊本県・福岡県のお住まいが対象です」
- 注釈：「LINEでお住まいの市町村を1つ送るだけ。営業電話はしません。」
- リンク先：LINE URL `https://lin.ee/UmuaYxJ`（将来的にエリア判定専用フローに分けることも可）
- 視覚スタイル：terra アウトライン枠＋「30秒」を大きく serif で表示。LP的な派手なボタンにはしない

### 1-4. 本CTA枠の差し替え（h-5直後／h-7／h-9）

LINE後3行コピー（カモシダ最終案・ユカリ修正反映：前向き起点）：
1. 写真を一枚送るだけで、概算と工期をお返しします。
2. 書類を集める前に動かないでください。サッシの型番を見ないと正しい金額は出ません。
3. 他社の見積もりが手元にある方も、そのまま送ってください。

各CTA枠の文言：
- h-5直後：「うちの窓だと、いくらくらい？」＋ボタン「LINEで写真を送る」
- h-7（費用工期直後）：「費用感が見えたら、次は写真を1枚。」＋ボタン「LINEで概算を聞いてみる」
- h-9（代表メッセージ後）：「『西日で困っているのですが』。そのひと言から、ぜひ。」＋ボタン「LINEで無料相談する」＋電話番号

担当者明示（ユカリ提案）：「担当：フクシマ建材 福嶋／返信は当日中、遅くとも翌営業日の午前まで」を各CTA下に明示。

### 1-5. sticky-line アンカー変更

## 2. 実装範囲外（公開前に別タスクで潰す）

ペルソナ会議で全員が合意した最優先事項だが、本LP実装の範囲外。**公開前に運用側で確定**：

### 2-1. LINE一次返信スクリプト3本（必須）
- スクリプト1：マイクロCV経由（エリア確認）— 対象エリアの即答＋写真依頼
- スクリプト2：本CV経由（写真送信前）— 写真依頼＋型番ガイド
- スクリプト3：本CV経由（写真送信後）— 概算返信テンプレ＋次の一手

### 2-2. 返信時間KPI設定
- 10分以内返信を目標（シノミヤ／キド合意）
- 担当者の顔写真＋名前を返信冒頭に表示（ユカリ提案）

### 2-3. 自動返信メッセージのコピー（カモシダ R5案）
> ご登録ありがとうございます。窓の写真を一枚、このトークに送ってください。サッシのメーカーシールが写っていれば、補助金対象かこちらで調べてお返事します。返信は当日中、遅くとも翌営業日の午前です。担当はフクシマ建材の福嶋です。

## 3. 公開後A/Bテストで判定する項目

- **マイクロCV設置が本CVを食うか純増か**（純増0.4pt以上が条件・岡村風指摘）
- 流入別CVR台帳の整備（広告/オーガニック/LINE/紹介/リスティング 5経路週次記録・シノミヤ提案）
- マイクロCV経由→本CV（現地調査）到達の追跡計測（岡村風提案）

## 4. 数字の見立て（討議結果）

| 指標 | 現行（v3想定） | 改修後（v4想定） |
|---|---|---|
| 全体CVR | 1.2-1.8% | 1.3-1.8%（流入別加重・キド審判で「鵜呑み禁」） |
| 読了率（最終CTA到達） | 18-22% | 25-28%（折りたたみ＋目次効果） |
| 月予算30万・流入2000での LINE登録件数 | 24-36件 | 26-36件 |
| CPA中央値 | 8000-12000円 | 8000-12000円（流入別差で振れる） |
| LINE登録→商談化 | 30%（推定） | 45%目標（運用側改善・LP範囲外） |

流入別レンジ：リスティング1.2-2.0% / インスタ0.4-0.8%（インスタはコピー直しが効きやすい）

## 5. 撤退条件

- 改修後30日でLINE着地からの一次返信率<70% → マイクロCVもA/Bも止めて運用設計に戻す（**最終ライン・キド裁定**）
- 改修後CVRがv2型を下回り続ける → 記事型骨格解体・LP寄り6:4へ（岡村風）
- マイクロCV経由のLINE登録→本CV到達が既存導線の半分未満 → マイクロCV撤去（岡村風）
- 90日加重CVRが改修前比+0.3pt未満 → 集客側へ予算振り替え（シノミヤ）
- LINE登録後の写真送信到達<20% → コピー起点主張全部捨てる（カモシダ）

---

## 6. 守るべき一線

- 記事型7割LP3割の比率は維持（マイクロCV増設で6:4に振れないように注意）
- 「分かりやすくお話しします」は説明姿勢の宣言で、記事を広告に戻すスイッチ → 削除確定
- h-1 冒頭一文で「広告で見た悩みの続きが、ここに書いてある」と0.5秒で錯覚させる
- LINE本CTAを h-4（認知段階）には置かない → マイクロCVのみが正解（岡村風・カモシダ）

---

## 7. 受け入れ条件

- v3 と並べて v4 を見たとき、改修点（タイトル句点・リード末・h-4見出し・折りたたみ・マイクロCV・本CTA枠の3行）がすべて反映されている
- 製品名「スタイルシェード」初出位置は v3 同様の中盤（マイクロCV後・h-5内）を維持
- マイクロCV ボタンは LP的な派手さを出さず、記事の続きの顔をしている
- LINE後3行が本CTA枠の中央に明示されている
- 担当者名と返信時刻目安が CTA下に明示されている
- グランドルール（井桁・アスタリスク禁止）コンプライアンス
